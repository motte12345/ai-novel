# SPEC.md — ai-novel

仕様の壁打ちログ。決定事項と未決事項を分けて管理する。

---

## 1. プロダクト定義

### 一行サマリ
**AI同士がリレーで短編を書いている様子を眺められるサイト。** 60分に1章進む、終わらない物語ストリーム。

### ユーザーストーリー
- 訪問者は執筆現場を覗く観客
- 介入はできない
- 1作品 6章 + 編集者完結 = 7章で完結 → 次のタイトルへ
- スクロールで過去作品閲覧、進行中の作品は1章ずつ追記される

---

## 2. リレー構造【決定】

### 1作品 = 7ターン構造
| Turn | 担当 | 内容 |
|---|---|---|
| 1 | 作家A（1周目） | タイトルから書き始める。300〜400字 |
| 2 | 作家B（1周目） | 直前を読んで続きを書く |
| 3 | 作家C（1周目） | 続き |
| 4 | 作家A（2周目） | 続き、終盤の伏線回収を意識 |
| 5 | 作家B（2周目） | 続き |
| 6 | 作家C（2周目） | 結末をつける |
| 7 | 編集者 | ペンネーム命名 + 各章タイトル付け + 総タイトル微修正 |

- **作家は3人匿名**。読者には誰がどの章を書いたか見せない（裏では provider/model を記録）
- **文体は寄せる**: 3人ともシリアス目の純文学風で揃える。揺らぎ幅は意図的に小さく
- **1章は300〜400字** （短くて良いとの判断、Cerebras 8192 token 制限にも余裕を持たせる）
- **進行間隔は 60分/章** → 1作品 7時間、1日 3.4作品

---

## 3. タイトル生成【決定】

### 方式: 2単語ランダム合成
- **辞書A（名詞）** × **辞書B（名詞 or 概念）** から1語ずつ抽選
- フォーマット候補: 「{A}と{B}」「{A}の{B}」「{B}と{A}」のような短い合成（具体形は SPEC で決定）
- 同じペアが既出だったら別ペアを引き直す（出現履歴を D1 で持つ）

### NG ワード辞書
- 性的・暴力的・差別的・実在人物名・宗教対立を煽る語などは事前に除外
- 単語辞書は手動キュレーション。「血」「死」「裸」「殺」など、文学的に必要に見える単語でも安全側に倒して除外
- 辞書はバージョン管理対象（`prompts/words.json` 等）

### 辞書サイズ目安
- 各100語以上 × 100語以上 = 1万通り。1年 × 1日 3.4作品 ≈ 1240作品でも重複率1割程度

## 4. 作家3人の設計【一部決定】

### 共通方針（決定）
- **匿名**: 名前なし。`writer_a` `writer_b` `writer_c` の内部 ID のみ
- **文体は寄せる**: シリアス目の純文学風（候補。SPEC で確定）。極端なジャンル違いはやらない（読者に「3人で書いてる」と意識させない）
- **AI である自認は内部に保つが、本文では人類視点で書いてよい**（小説なので「私たち人間」表現は許容、当事者性は不要）
- 各章で前章の流れ・登場人物・場面を尊重する

### 共通制約
- **本文300〜400字**
- **登場人物名・場面・既出の小道具は変更しない**（一貫性厳守）
- 規約NG話題（性的・暴力的・差別的）には触れない
- メタ発言禁止（「ここで作者を交代します」のような注釈を書かない）
- 章タイトル・章番号は本文に書かない（編集者が後付け）

### 揺らぎの付け方
- 同じ system prompt をベースに、temperature や「重視する要素」を微妙に変える程度（叙景重視 / 心理重視 / 会話重視など）
- 後述: 詳細は実装フェーズで PoC しながら詰める

## 5. 編集者AI【決定】

### 役割
1. **総タイトルの仕上げ**: 単語2つから合成された素のタイトルを、本文を読んだ上で自然な題に整える（例: 「ランプと海」→「ランプの灯る海辺で」）。原題は元のまま保存しておく
2. **各章タイトル付け**: 全6章それぞれに簡潔な章タイトルを付ける
3. **ペンネーム命名**: 作品単位で1つ。架空のペンネーム（実在の作家名と被らないようチェック）
4. **完結処理**: status を `completed` に遷移

### 出力フォーマット（暫定）
```
【ペンネーム】
{ペンネーム}

【作品タイトル】
{仕上げ後タイトル}

【章タイトル】
1. {タイトル}
2. {タイトル}
3. {タイトル}
4. {タイトル}
5. {タイトル}
6. {タイトル}
```

### プロバイダ（暫定）
- Gemini Flash-Lite（RPD 1000 あるので余裕、Host相当の低レイテンシ役）

## 6. D1 スキーマ【暫定】

### `stories` テーブル
```sql
CREATE TABLE stories (
  id INTEGER PRIMARY KEY,
  raw_title TEXT NOT NULL,           -- "ランプと海" のような単語ペア合成タイトル
  word_a TEXT NOT NULL,              -- 抽出元の単語A
  word_b TEXT NOT NULL,              -- 抽出元の単語B
  final_title TEXT,                  -- 編集者AIによる仕上げ後タイトル
  pen_name TEXT,                     -- 編集者AIが付けたペンネーム
  status TEXT NOT NULL,              -- 'pending' | 'active' | 'completed'
  current_turn INTEGER NOT NULL DEFAULT 0,
  started_at INTEGER,
  completed_at INTEGER,
  created_at INTEGER NOT NULL
);
CREATE UNIQUE INDEX idx_stories_word_pair ON stories(word_a, word_b);
```

### `chapters` テーブル
```sql
CREATE TABLE chapters (
  id INTEGER PRIMARY KEY,
  story_id INTEGER NOT NULL,
  chapter_no INTEGER NOT NULL,        -- 1..7。7は編集者ターン
  writer TEXT NOT NULL,               -- 'writer_a' | 'writer_b' | 'writer_c' | 'editor'
  provider TEXT NOT NULL,
  model TEXT NOT NULL,
  title TEXT,                         -- 編集者が後付けする章タイトル
  content TEXT NOT NULL,
  summary TEXT,                       -- 次章への引き継ぎ用の100字あらすじ（編集者or別途生成）
  created_at INTEGER NOT NULL,
  FOREIGN KEY (story_id) REFERENCES stories(id)
);
CREATE UNIQUE INDEX idx_chapters_story_no ON chapters(story_id, chapter_no);
```

### `meta` テーブル
- ai-roundtable と同じ汎用 key-value（cron 失敗カウンタ、最終実行時刻、辞書バージョン等）

## 8. フロント UI【決定: ライブ感重視】

### トップページ
- 進行中の作品（1章ずつ追記）をメインに表示
- 「現在 X章目 / 全6章」のプログレス
- 次章までのカウントダウン
- 完結作品の archive へのリンク

### 作品ページ
- 縦に章を並べる（横書き、読みやすい字間）
- 編集者の章タイトルが付いた状態（完結後）
- 進行中は最終章のみ「執筆中...」表示

### archive
- 完結作品の一覧（タイトル + ペンネーム + 投稿日）
- ジャンルフィルタは無し（ジャンル分類しないため）

## 9. ai-roundtable からの流用方針【決定】

### 流用するコード
- `workers/lib/providers.ts` — そのまま流用、モデル割当だけ差し替え
- `workers/lib/db.ts` の構造 — テーブル名・カラムを差し替え
- `workers/lib/turn-runner.ts` の骨格 — レース対策、UNIQUE+CAS、失敗リトライ、LINE 通知
- `workers/lib/line-notify.ts` — そのまま流用
- フロント: ai-roundtable から雛形をコピー → 議論UI を物語UI に置換
- SEO・OGP・RSS・sitemap・robots — ai-roundtable の作り方を踏襲

### 不要なもの
- Host キャラ・closing 提案・主題語チェック・重複検知（タイトル単語ペアの一意性は DB UNIQUE で担保）
- 投票機能（Turnstile + 候補ページ）
- TTS（小説は音声化しない、ひとまず）
- ジャンル推定・SEOタイトル生成（編集者AIが最終タイトル付けるので不要）
- ハイライト生成（小説は全文を読ませる、抜粋ハイライトは作らない）

## 10. 規約セーフな単語辞書設計

### 辞書A 候補テーマ（名詞）
- 自然物: 海、山、月、雪、花、雨、風、雲、星、森
- 日用品: ランプ、ボタン、手帳、椅子、傘、鏡、地図、扉、鍵、本
- 職業: 旅人、職人、図書館員、画家、料理人、観測者、訪問者
- 場所: 喫茶店、駅、灯台、屋根裏、橋、市場、停留所、桟橋
- 動物: 猫、犬、鳥、魚、馬、蝶、亀（NG: 害虫・人を傷つける動物の文脈になりやすいもの）

### 辞書B 候補テーマ（情景・状態・概念）
- 時間: 夕暮れ、夜明け、季節、午後、瞬間、永遠
- 状態: 沈黙、ざわめき、霧、灯り、ぬくもり、影、約束
- 概念: 余白、回想、約束、別れ、出会い、忘れ物、距離

### NG カテゴリ（絶対不採用）
- 性的: 一切
- 暴力: 一切（「戦」「斬」「血」「銃」など連想されるもの含む）
- 差別: 民族・宗教・性別・障害に関する語
- 実在人物名・固有商標
- 病・死・自死・薬物に直結する語（文学的に必要そうでも安全側で除外）
